# NewTech Payment Processor — рабочие задачи

Источник требований: `docs/source-task-notes.md` (заметки из Obsidian).

## В работе (по порядку)
1. [x] Исправить приоритет Архива счетов: точное совпадение контрагента и номера счета переносит объект, проект, статью, ответственного и ссылку; не подменять `Рампа` на `Тунгус`.
2. [x] Исправить распознавание счета и ПП: сохранять полный номер счета и нормализованный контрагент, чтобы ПП находился по номеру/сумме/назначению.
3. [x] Исправить наличные: не назначать наличным операции банк Сбербанк/Альфа/Точка; наличка должна попадать один раз, включая конвертацию +348 000.
4. [x] Не объединять зарплатные строки: три записи зарплаты остаются тремя строками. Объединять можно только банковскую комиссию по согласованному правилу.
5. [x] Подсвечивать только FinTablo-операции, которых нет среди MAX/чата: доходы бледно-зеленым, расходы бледно-оранжевым.
6. [x] Исправить типы данных Google Sheets: дата — настоящая дата с форматом `dd.MM.yyyy`, сумма — число с десятичной запятой; не выгружать их строками.
7. [x] Проверить фильтр `Итоговая ИС`: только операции ИнвестСтроя и его счетов, без ПСК Ньютек/ИП Мочалова/Сбербанка.
8. [ ] Восстановить ссылки только в рабочем Drive `pcknew.tech@gmail.com`; личный Drive не использовать. Отсутствующие файлы искать в чатах и ставить в очередь загрузки.
9. [ ] Восстановленные ссылки выделять зеленым жирным шрифтом для ручного переноса в рабочую таблицу.
10. [ ] Проверить правило `UPay`: ответственный берется из текста сообщения UPay, сам UPay ответственным не является.
11. [ ] Поддержать лист `Ручная сверка`: сравнивать ручные исправления со скриптовой итоговой и переносить подтвержденные правила в обработчик.

## Следом
- [ ] Устойчивая очередь загрузки файлов: повторные попытки, статус каждой задачи, понятный отчет в Telegram.
- [ ] Отчет по FinTablo: ID транзакции, дата, сумма, контрагент и конкретная причина, если запись не перенесена.
- [ ] Повторная проверка всех данных с апреля после внесения правил.

## Правило источников
- MAX/чат — сообщения, наличка и счета.
- FinTablo — банковские транзакции и дополнительные операции (комиссии, проценты, кредиты, приходы).
- Архив счетов и ручная сверка — каноничная классификация при точном совпадении счета.
