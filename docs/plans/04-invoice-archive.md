# Архив счетов

## Цель

Создать архив счетов из MAX: сохранить файлы счетов, подписи сотрудников, нормализованные поля и ссылку на файл в Google Drive.

## Текущий статус

Локальная часть может быть реализована до Google-доступов: парсер подписей, словари нормализации, черновой архив в Excel/CSV/JSON и тесты.

## Важные решения

- Архив счетов ведется в одной общей Google таблице.
- Файлы на Drive должны попадать в существующую структуру: `Объект / Год / Месяц / файл`.
- Значения из MAX проходят через словарь нормализации.
- Если объект не найден или не совпал со справочником, статус строки `Нужно разобрать`.
- OCR в первой версии не обязателен: сканы сохраняются, но данные из них не извлекаются.
- Сообщения MAX без файла тоже могут создавать строки архива, если это понятная операция. Первая поддержанная форма: `ИП Мочалов`, строки `зп - сумма` и `налоги - сумма`.
- При повторной сверке источником документа является только файл по Drive ID из
  колонки `Google Drive ссылка` конкретной строки. Локальный файл не выбирается
  по отображаемому имени.
- Из чата пересчитываются только `Объект`, `Проект`, `Статья бюджета`,
  `Ответственный` и `Назначение`.
- Дата и тип оплаты, контрагент, номер счета, сумма, Drive-ссылка и статус оплаты
  при сверке сообщений не изменяются.
- Текст и файл в одном `body`, а также `reply/forward`, считаются точной связью.
  Для отдельных сообщений рассматриваются подписи и до, и после файла.
- Несколько допустимых пар означают неоднозначность: существующие чатовые поля
  и `Статус разбора` сохраняются без изменений. Автоматическая запись разрешена
  только для доказанной однозначной связи.
- Сопоставление выполняется взаимно-однозначно для группы сообщений, а не
  жадным выбором ближайшей подписи.
- Перед изменением Google Sheet формируется локальный отчет `строка -> Drive ID
  -> MAX ID -> уровень надежности`; после записи измененные ячейки перечитываются.

## Колонки архива

`Дата MAX`, `Режим`, `Чат`, `Автор`, `Имя файла`, `Тип файла`, `Контрагент`, `Номер счета`, `Дата счета`, `Год счета`, `Объект`, `Проект`, `Статья бюджета`, `Ответственный`, `Назначение`, `Сумма`, `Google Drive ссылка`, `MAX message_id`, `MAX file_id`, `Статус`.

## Что нужно от пользователя

- ID или ссылка Google таблицы архива счетов.
- Название листа архива.
- OAuth/Google Cloud client для локальной программы.
- Корневая папка Google Drive с объектами.
- 5-10 примеров реальных путей Drive.
- Список объектов и варианты, как сотрудники пишут их в MAX.
- Список ответственных и частые сокращения.

## Проверки

- Проверить парсинг подписей MAX с точками, переносами и разным регистром.
- Проверить нормализацию объектов, проектов, статей и ответственных.
- Проверить статус `Нужно разобрать`.
- Проверить отсутствие дублей при повторном запуске.
- Проверить текстовую операцию без файла: `ИП Мочалов / зп / налоги`.
- Проверить подпись перед файлом и после файла.
- Проверить неоднозначную группу из нескольких файлов и нескольких подписей.
- Проверить, что содержимое документа читается по Drive ID строки и что
  документные поля не изменяются при пересборке чатовых полей.
