# Правки обработчика ПП

## Цель

Зафиксировать внесенные уточнения по полям платежных поручений, правилам и интерфейсу проверки.

## Текущий статус

- Правила из Power Query перенесены в `rules.json`.
- `Номер счета` не может быть `НДС`; если счет не найден, используется `б/сч`.
- Назначение платежа обрезается до служебных хвостов про счет и НДС.
- Банк определяется как банк плательщика с учетом плательщика, например `б/н ИП Мочалов Сбер`.
- Контрагент и назначение собираются из нескольких строк.
- Встроенная таблица поддерживает фильтр, сортировку, редактирование и выпадающие списки из справочника.

## Важные решения

- `rules.json` остается запасным механизмом для постоянных офисных платежей.
- Основная логика распределения по объектам должна перейти к архиву счетов и подписи MAX.
- Пустые поля лучше, чем фиктивные значения вроде `проверить`.

## Проверки

- Проверять проблемные файлы с многострочным контрагентом.
- Проверять счета вида `СЧЕТ № 19971`.
- Проверять банк плательщика для ИП Мочалов, Точка, Альфа, Сбербанк.
